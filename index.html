一、标准制定背景

为支撑分子性质预测、化学反应建模、谱图解析、分子表征学习、相似性检索、产物预测、 反应条件预测、逆合成分析、产率预测以及多模态大模型训练等任务，本项目构建了覆盖分子结构、 理化性质、化学反应、反应条件、产物结构、产率信息以及核磁共振谱图信息的多模态化学数据库。

结构标准化

统一字段与格式

采用统一字段命名、统一数据类型和统一存储格式，支持不同来源数据在同一规范下管理和调用。

模态可关联

跨模态映射

通过 SMILES、canonical SMILES、InChIKey、reaction SMILES 等关键字段建立关联关系。

质量可校验

自动化评价

支持对字段完整性、格式规范性、数据类型、模态对应关系和有效性状态进行自动化检查。

二、数据类型与规模

数据库由分子数据、化学反应数据和谱图数据三部分构成，合计 538,046,852 条有效数据。

Molecule Data

分子数据

400,000,000 条。描述单个化合物的结构与理化性质，常用于分子表征、性质预测和相似性检索。

Reaction Data

化学反应数据

32,281,977 条。描述反应输入、反应条件、产物结构、数据来源和任务标签。

Spectrum Data

谱图数据

105,764,875 条。包含分子信息和核磁共振谱图信息，支持谱图预测和结构解析。

三、核心字段标准

分子数据不包含三维结构字段；三维结构 token 属于化学反应数据中的反应物、试剂或产物结构表示。

3.1 分子数据字段

字段名称字段含义主要用途
smiles分子的 SMILES 线性结构表示，用 ASCII 字符串编码原子、键、支链和环等连接关系。分子结构建模、检索与相似性分析
mol_wt分子量，通常以 g/mol 计。性质预测与分子筛选
logp辛醇/水分配系数的对数，反映分子疏水性或脂溶性。疏水性、脂溶性与溶解性分析
tpsa拓扑极性表面积，反映极性原子贡献的表面积。极性、渗透性与相互作用分析
hbd氢键供体数量。分子相互作用与性质建模
hba氢键受体数量。氢键、极性特征与性质建模

3.2 化学反应数据字段

字段名称字段含义主要用途
dataset数据集名称，例如 USPTO_MIT、USPTO_Condition、USPTO_50k、Reaxys、pistachio、Buchwald-Hartwig 等。数据追溯、任务划分与来源区分
reactant_1d_smiles反应物一维 SMILES。反应物结构输入
reactant_3d_smiles反应物三维结构 token。反应物空间构象输入
reagent_1d_smiles试剂、溶剂、催化剂、碱等一维 SMILES。反应条件建模
reagent_3d_smiles试剂或条件分子的三维结构 token。条件分子空间结构建模
product_1d_smiles产物一维 SMILES。产物预测与结构表示
product_3d_smiles产物三维结构 token。产物空间构象建模
reaction_smiles完整反应 SMILES，格式为 reactants>reagents>products。反应级别统一表示
yield / target反应产率或模型预测目标。产率预测与监督学习标签

3.3 谱图数据字段

字段名称字段含义主要用途
smiles原始 SMILES 分子结构字符串。分子结构输入
canonical_smiles标准化 SMILES。去重、检索与模型输入
inchikey分子唯一标识符。跨数据库检索与分子匹配
H_nmr预测 1H NMR 化学位移列表，单位 ppm。氢谱预测与解析
C_nmr预测 13C NMR 化学位移列表，单位 ppm。碳谱预测与解析
nmr_predict完整 NMR 预测序列，单位 ppm。统一谱图建模与预测
atom_index原子类型或原子序数列表。原子级谱图关联
equi_class等价原子类别编号。等价原子识别与谱峰归属

四、数据质量控制与评价方法

质量控制主要围绕有效数据总量、标准符合率、字段完整性、格式规范性和模态对齐关系展开。

有效数据总量

Total Valid Records，记为 Ntotal，表示经过清洗去重且状态标记为有效的数据记录总数。

Ntotal = Σ Count(Tablei), i = 1, 2, ..., M

标准符合率

Standard Compliance Rate，记为 Rcompliance，表示随机抽样数据中完全符合标准规范的样本比例。

Rcompliance = Npass / Nsample × 100%

抽样校验流程

通过数据库管理工具或统计脚本连接至数据存储环境。
针对核心数据表执行全量去重计数操作。
统计唯一有效数据总量，验证数据规模是否达标。
采用分层随机抽样方法抽取样本，例如 1000 条。
按照多模态数据标准规范对样本进行自动化校验。
检查字段完整性、格式规范性、数据类型和模态关联关系。
计算有效数据总量和标准符合率，并输出数据质量评估报告。

五、适用任务

本数据库可支撑从分子级别到反应级别、谱图级别的多任务建模，也可作为多模态大模型训练与智能体工具调用的数据基础。

分子性质预测 分子相似性检索 分子表征学习 虚拟筛选 反应产物预测 反应条件预测 逆合成分析 反应产率预测 核磁谱图预测 核磁谱图解析 分子-谱图多模态对齐 多模态大模型训练

六、GitHub Pages 发布说明

将本页面文件命名为 index.html，上传到 GitHub 仓库根目录，然后在仓库 Settings → Pages 中选择部署分支，即可通过 GitHub Pages 发布为网页。

推荐仓库结构

multimodal-chemical-database/ ├── index.html ├── README.md └── docs/ └── multimodal_data_standard.md

发布步骤

新建 GitHub 仓库并上传 index.html；进入 Settings → Pages； Source 选择 Deploy from a branch，Branch 选择 main 和 /root； 保存后等待部署完成。
