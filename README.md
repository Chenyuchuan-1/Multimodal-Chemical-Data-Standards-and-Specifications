# 多模态化学数据库标准说明

## 项目简介

本项目构建了一个面向化学与材料智能研发的多模态化学数据库，覆盖分子数据、化学反应数据和谱图数据三类核心数据资源。数据库可用于分子表征学习、性质预测、相似性检索、化学反应预测、反应条件预测、逆合成分析、产率预测、核磁谱图解析以及多模态大模型训练等任务。

数据库围绕结构化化学数据进行统一整理，建立了字段命名、数据格式、模态关联和质量校验标准，为后续模型训练、平台调用、数据检索和智能体工具集成提供标准化数据基础。

## 数据规模

当前多模态数据库共包含 **538,046,852** 条有效数据，具体构成如下：

| 数据类型 | 数据量 |
|---|---:|
| 分子数据 | 400,000,000 |
| 化学反应数据 | 32,281,977 |
| 谱图数据 | 105,764,875 |
| 合计 | 538,046,852 |

所有数据均经过清洗、去重和有效性标记处理，统计结果以去重后的有效数据为准。

## 数据类型说明

### 1. 分子数据

分子数据用于描述单个化合物的结构与理化性质，常用于分子表征、性质预测和相似性检索。

#### 字段说明

| 字段名 | 含义 |
|---|---|
| `smiles` | 分子的 SMILES 线性结构表示，用 ASCII 字符串编码原子、键、支链和环等连接关系 |
| `mol_wt` | Molecular weight，分子量，通常以 g/mol 计，用于描述分子整体质量 |
| `logp` | 辛醇/水分配系数的对数，反映分子疏水性或脂溶性 |
| `tpsa` | Topological polar surface area，拓扑极性表面积，反映极性原子贡献的表面积 |
| `hbd` | Hydrogen bond donors，氢键供体数量 |
| `hba` | Hydrogen bond acceptors，氢键受体数量 |

#### 典型应用

- 分子性质预测
- 分子表征学习
- 相似性检索
- 分子空间分析
- 虚拟筛选
- 分子生成结果评估

---

### 2. 化学反应数据

化学反应数据用于描述反应输入、反应条件、产物结构、数据来源和任务标签，常用于产物预测、条件预测、逆合成、反应表示学习和产率预测等任务。

#### 数据来源

当前反应数据主要包含以下数据集来源：

- `USPTO_MIT`
- `USPTO_Condition`
- `USPTO_500_MT`
- `USPTO_50k`
- `SMol-Reactions-RS`
- `SMol-Reactions-FP`
- `Reaxys`
- `pistachio`
- `Buchwald-Hartwig`

#### 通用字段说明

| 字段名 | 含义 |
|---|---|
| `dataset` | 数据集名称，用于区分反应数据来源和任务类型 |
| `reactant_1d_smiles` | 反应物的一维 SMILES，描述参与反应的起始分子结构 |
| `reactant_3d_smiles` | 反应物的三维结构 token 表示，包含空间构象信息 |
| `reagent_1d_smiles` | 试剂、溶剂、催化剂、碱等条件分子的一维 SMILES |
| `reagent_3d_smiles` | 试剂或条件分子的三维结构 token 表示 |
| `product_1d_smiles` | 产物的一维 SMILES，描述反应生成物结构 |
| `product_3d_smiles` | 产物的三维结构 token 表示 |

#### 配体相关字段

| 字段名 | 含义 |
|---|---|
| `smiles` | 配体或小分子的一维 SMILES |
| `smiles_3d` | 配体或小分子的三维结构 token 表示 |

#### Buchwald-Hartwig 反应字段

| 字段名 | 含义 |
|---|---|
| `source` | 模型输入序列，表示完整反应信息，包含芳基卤化物、胺、催化剂、配体、碱、溶剂和产物等 |
| `target` | 模型预测目标，通常为反应产率数值 |
| `reaction_smiles` | 完整反应 SMILES，格式为 `reactants>reagents>products` |
| `reactants` | 反应物，例如芳基卤化物和胺 |
| `reagents` | 反应条件，例如 Pd 催化剂、配体、碱、溶剂等 |
| `products` | 反应产物 |
| `yield` | 反应产率，即该反应条件下目标产物的实验产率数值 |

#### 典型应用

- 反应产物预测
- 反应条件预测
- 逆合成分析
- 反应产率预测
- 反应表示学习
- 催化反应条件优化

---

### 3. 谱图数据

谱图数据包含分子信息和核磁共振谱图信息，主要用于核磁谱图预测、谱图解析、分子识别和分子-谱图多模态对齐。

#### 字段说明

| 字段名 | 含义 |
|---|---|
| `smiles` | 原始 SMILES 分子结构字符串，用线性文本表示分子的原子、键、环和支链等连接关系 |
| `canonical_smiles` | 标准化后的 SMILES，同一个分子通常对应唯一或更稳定的表示，便于去重、检索和模型输入 |
| `inchikey` | InChIKey 分子唯一标识符，用于跨数据库检索同一个化合物 |
| `H_nmr` | 预测的氢谱 1H NMR 化学位移列表，单位为 ppm |
| `C_nmr` | 预测的碳谱 13C NMR 化学位移列表，单位为 ppm |
| `nmr_predict` | 完整 NMR 预测值序列，单位为 ppm |
| `atom_index` | 原子类型或原子序数列表 |
| `equi_class` | 等价原子类别编号，编号相同表示这些原子在分子对称性或化学环境中属于同一等价类 |

#### 典型应用

- 1H NMR 谱图预测
- 13C NMR 谱图预测
- 核磁谱图解析
- 分子结构确认
- 分子-谱图多模态对齐
- 核磁智能解谱模型训练

## 数据标准化原则

本数据库遵循以下标准化原则：

1. **字段命名统一**：对分子、反应和谱图数据采用统一字段命名，减少不同来源数据之间的歧义。
2. **数据类型明确**：字符串、数值、列表和序列字段均需满足固定数据类型要求。
3. **结构表示规范**：分子结构采用 SMILES 或 canonical SMILES 表示，反应结构采用 reaction SMILES 表示。
4. **模态关联清晰**：分子、反应和谱图数据通过 SMILES、canonical SMILES、InChIKey、reaction SMILES 等字段进行关联。
5. **去重统计明确**：数据规模统计以清洗、去重并标记为有效的数据为准。
6. **质量可自动校验**：数据字段完整性、格式规范性和模态对应关系均可通过自动化脚本进行检查。

## 数据质量校验

数据库质量评价主要包括有效数据总量和标准符合率两个指标。

### 有效数据总量

有效数据总量表示数据库中经过清洗去重且状态标记为有效的数据记录总数。

```text
Ntotal = Σ Count(Tablei), i = 1, 2, ..., M
```

其中：

- `M` 为数据分表数量；
- `Count(Tablei)` 为第 `i` 个数据表中的有效记录数量；
- `Ntotal` 为有效数据总量。

### 标准符合率

标准符合率表示随机抽样数据中完全符合多模态数据标准规范的样本比例。

```text
Rcompliance = Npass / Nsample × 100%
```

其中：

- `Nsample` 为抽样样本总数；
- `Npass` 为校验通过的样本数量；
- `Rcompliance` 为标准符合率。

当样本同时满足 Schema 定义、字段类型、字段完整性、格式规范性和模态关联关系要求时，方可计入 `Npass`。

## 抽样校验流程

数据库规模与多模态标准符合度可按照以下流程进行验证：

1. 通过数据库管理工具或统计脚本连接至数据存储环境；
2. 针对核心数据表执行全量去重计数操作；
3. 统计唯一有效数据总量，验证数据规模是否达标；
4. 采用分层随机抽样方法从数据库中抽取样本，例如 1000 条；
5. 按照《多模态数据标准规范》对样本进行自动化校验；
6. 检查字段完整性、格式规范性、数据类型和模态关联关系；
7. 计算有效数据总量和标准符合率；
8. 输出数据质量评估报告。
